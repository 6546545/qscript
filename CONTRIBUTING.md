## Contributing to QScript

Thank you for your interest in contributing!

### Getting Started

- Read `README.md` and the design docs under `docs/` to understand the project goals.
- Browse open issues or propose new ones if you have ideas or find bugs.

### Contribution Workflow

1. Fork the repository and create a feature branch.
2. Make your changes in small, focused commits.
3. Ensure code builds and passes basic checks.
4. Open a pull request describing the motivation and approach.

### Areas to Contribute

- Language design and documentation.
- Compiler frontend (lexer, parser, type checker).
- IR and backends (classical and quantum).
- Tooling and developer experience.
- Examples, tests, and tutorials.

Please keep discussions respectful and constructive, following the `CODE_OF_CONDUCT.md`.
